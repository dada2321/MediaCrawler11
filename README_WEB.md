# MediaCrawler Web 管理平台

## 🌟 功能特性

MediaCrawler Web 管理平台为原有的命令行爬虫工具提供了现代化的 Web 界面，让您能够更加便捷地管理和分析爬取的数据。

### 📊 主要功能

1. **数据概览仪表板**
   - 实时统计数据展示
   - 各平台数据分布图表
   - 最近爬取活动时间线
   - 系统运行状态监控

2. **爬虫控制中心**
   - 图形化启动/停止爬虫
   - 实时任务状态监控
   - 灵活的参数配置
   - 任务历史记录查看

3. **数据管理系统**
   - 多平台数据统一查看
   - 强大的搜索和筛选功能
   - 帖子详情和评论查看
   - 数据导出（CSV格式）

4. **配置管理界面**
   - 可视化配置编辑
   - 配置文件导入/导出
   - 分类配置管理
   - 实时配置验证

## 🚀 快速开始

### 1. 安装依赖

```bash
# 安装 Web API 依赖
pip install -r web_requirements.txt

# 安装前端依赖（可选，仅开发时需要）
cd frontend
npm install
```

### 2. 启动 Web 服务

```bash
# 启动 Web API 服务器
python web_api.py
```

服务启动后，访问以下地址：
- **管理界面**: http://localhost:8000
- **API 文档**: http://localhost:8000/docs

### 3. 使用说明

1. **首次使用**：
   - 访问管理界面
   - 在"配置管理"页面设置基本参数
   - 在"爬虫控制"页面启动第一个任务

2. **监控爬虫**：
   - 在"数据概览"查看整体统计
   - 在"爬虫控制"监控任务状态
   - 查看实时日志和错误信息

3. **数据分析**：
   - 在"数据管理"浏览爬取结果
   - 使用搜索和筛选功能
   - 导出数据进行进一步分析

## 🏗️ 架构说明

### 技术栈

- **后端**: FastAPI + Python
- **前端**: React + TypeScript + Ant Design
- **数据库**: SQLite / MySQL
- **图表**: ECharts
- **构建**: Vite


## 🔧 高级配置

### 自定义端口

```bash
# 自定义 API 服务端口
uvicorn web_api:app --host 0.0.0.0 --port 8080
```

### 生产环境部署

1. **构建前端**：
```bash
cd frontend
npm run build
```

2. **启动生产服务**：
```bash
python web_api.py
```

### 数据库配置

Web 平台自动使用项目的数据库配置：
- SQLite: 自动创建 `data/mediacrawler.db`
- MySQL: 使用 `config/db_config.py` 中的配置

## 🔐 安全说明

- Web 服务默认仅绑定本地地址
- 生产环境建议配置反向代理
- 敏感配置信息不会在前端显示
- 支持 CORS 跨域访问控制

## 🐛 故障排除

### 常见问题

1. **端口占用**：
   - 修改 `web_api.py` 中的端口设置
   - 或使用 `--port` 参数指定其他端口

2. **数据库连接失败**：
   - 检查数据库文件权限
   - 确认 SQLite 文件路径正确

3. **前端资源加载失败**：
   - 确认已构建前端资源
   - 检查静态文件路径配置

4. **API 请求失败**：
   - 检查后端服务是否正常运行
   - 确认 API 端点地址正确

### 日志查看

Web API 服务会输出详细的访问日志和错误信息，有助于问题诊断。
